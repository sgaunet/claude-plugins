# Generate .github Directory Command

Instantly create a complete `.github` directory structure for Go projects with production-ready GitHub Actions workflows, Dependabot configuration, and sponsorship setup.

## Why This Command Exists

**Problem**: Setting up CI/CD for Go projects requires manually creating multiple workflow files, understanding GitHub Actions syntax, and configuring dependencies - a 30-60 minute process.

**Solution**: One-command generation that:
- Creates complete `.github` directory structure (6 files, 549 lines)
- Includes comprehensive workflows: linting, coverage, snapshot builds, releases
- Configures Dependabot for Go modules, Docker, and GitHub Actions
- Sets up GitHub Sponsors integration
- Preserves all customization comments for user control

## Content Generation Rules

**CRITICAL**: When generating any content (workflow files, commit messages, or any other output):
- **NEVER** add Claude Code attribution or co-authorship
- **NEVER** include phrases like "Generated by Claude", "Created with Claude Code", or similar
- **NEVER** add "Co-Authored-By: Claude" in commit messages or documentation
- All generated content should appear as if written by a human developer
- Focus on technical accuracy and providing working templates

### Phase 3.5: Auto-Detection & Substitution

**Detection Function:**

```bash
# Extract owner from git remote URL
detect_git_owner() {
    local remote_url=$(git remote get-url origin 2>/dev/null)
    [[ -z "$remote_url" ]] && return 1

    # SSH: git@github.com:sgaunet/repo.git → sgaunet
    if [[ "$remote_url" =~ git@[^:]+:([^/]+)/ ]]; then
        echo "${BASH_REMATCH[1]}"
        return 0
    fi

    # HTTPS: https://github.com/sgaunet/repo.git → sgaunet
    if [[ "$remote_url" =~ https?://[^/]+/([^/]+)/ ]]; then
        echo "${BASH_REMATCH[1]}"
        return 0
    fi

    return 1
}

# Substitute placeholder in FUNDING.yml
substitute_github_username() {
    local content="$1"
    local username="$2"

    # Escape special characters
    username=$(echo "$username" | sed 's/[&\/]/\\&/g')

    # Replace placeholder
    echo "$content" | sed "s|\[GITHUB_USERNAME\]|${username}|g"
}
```

**Detection Workflow:**

1. **Detect GitHub username:**
   ```bash
   GITHUB_USERNAME=$(detect_git_owner)
   ```

2. **Substitute in FUNDING.yml template (only if not in minimal mode):**
   ```bash
   if [[ "$minimalMode" != "true" ]]; then
       funding_template=$(substitute_github_username "$funding_template" "$GITHUB_USERNAME")
   fi
   ```

3. **Handle missing value:**
   - If `GITHUB_USERNAME` is empty: Preserve `[GITHUB_USERNAME]` placeholder with warning
   - Non-blocking: User can still create files and edit manually later

## Command Arguments

Arguments are parsed from the `argument` string parameter:

- **`--force`**: Skip confirmation prompt and overwrite existing files
  - Use when: Upgrading existing workflows to latest templates
  - Behavior: Creates backups? No (files are overwritten directly)

- **`--minimal`**: Generate only 3 essential files
  - Includes: dependabot.yml, linter.yml, release.yml
  - Excludes: FUNDING.yml, coverage.yml, snapshot.yml
  - Use when: Quick CI/CD setup without advanced features

- **`--dry-run`**: Preview files without writing to disk
  - Shows: File list, line counts, preview of content
  - Writes: Nothing
  - Use when: Reviewing what will be created before committing

**Parsing example:**
```
argument = "--force --minimal"
Split by space, check for each flag:
- forceMode = true
- minimalMode = true
- dryRunMode = false
```

## Error Handling

### Critical Errors (Exit Immediately)

**Not a Git repository:**
```
❌ Error: Not a git repository

This command requires a Git repository. Initialize one:
  git init
  git add .
  git commit -m "Initial commit"
```

**Not a Go project:**
```
❌ Error: go.mod not found

This command is for Go projects only. Initialize a Go module:
  go mod init github.com/username/project
```

**Command assets missing:**
```
❌ Error: Command assets directory not found

The go-specialist plugin may be corrupted. Try:
1. Reinstalling the plugin
2. Updating Claude Code to latest version
3. Reporting issue at: https://github.com/sgaunet/claude-plugins/issues
```

### Warning Errors (Continue with Others)

**Existing .github directory:**
- Without `--force`: Request user confirmation
- With `--force`: Overwrite silently

**Individual file write failure:**
- Log error message
- Continue with remaining files
- Report partial success at end

**Missing optional asset (e.g., FUNDING.yml):**
- Skip file with warning
- Continue with other files
- Note in final report

### Recovery Actions

**If mkdir fails:**
```
❌ Error: Failed to create .github directory

Check:
1. Current directory is writable
2. Sufficient disk space
3. No existing .github file (not directory) blocking creation

Current permissions: [show with ls -ld .]
```

**If all writes fail:**
```
❌ Error: Failed to write any workflow files

This may indicate:
1. Permission issues in .github directory
2. Disk full
3. File system mounted read-only

Check disk space: df -h .
Check permissions: ls -la .github/
```

## Template Customization Points

All template files include `# CUSTOMIZE:` comments marking required changes:

**FUNDING.yml:**
- `github: [YOUR_GITHUB_USERNAME]` → Replace with your username

**workflows/*.yml:**
- `go-version: stable` → Set to specific version (e.g., "1.24.x")
- `go-version: '>=1.24'` → Update minimum version
- Tag patterns in release.yml → Adjust semver format if needed

**dependabot.yml:**
- `interval: monthly` → Change to daily/weekly for active projects
- `open-pull-requests-limit: 10` → Adjust PR concurrency

**coverage.yml:**
- `limit-coverage: "70"` → Adjust coverage threshold
- Badge repository reference → Update if using fork
- `sed -i '/cmd/d'` → Remove if including cmd package in coverage

## Limitations

**This command does NOT:**
- Generate Taskfile.yml (too project-specific, create manually)
- Generate .goreleaser.yml (use `/goreleaser` command)
- Generate .golangci.yml (use `/linter` command)
- Modify existing workflow files intelligently (use `--force` to overwrite)
- Validate that workflows will work (requires proper setup)
- Configure GitHub repository settings (must be done via UI)

**Scope boundaries:**
- Only handles `.github/` directory structure
- Templates are opinionated (aligned with gitlab-backup project patterns)
- Assumes standard Go project structure
- Requires manual customization post-generation
