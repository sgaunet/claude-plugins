# Generate GoReleaser Configuration Command

Instantly create complete GoReleaser setup for Go projects with multi-architecture Docker builds, automated releases, and package distribution. This command converts the multi-step `goreleaser` skill into a one-command solution.

## Why This Command Exists

**Problem**: Setting up GoReleaser requires:
- Creating complex YAML configuration (~300 lines)
- Designing multi-stage Dockerfiles for cross-platform builds
- Configuring non-root container users
- Understanding ldflags, build matrix, Docker buildx
- Testing across multiple architectures
A 60-90 minute process requiring deep GoReleaser knowledge.

**Solution**: One-command generation that:
- Creates production-ready `.goreleaser.yml` (316 lines)
- Includes optimized `Dockerfile` for multi-arch builds
- Sets up non-root container user configuration
- Auto-detects project structure and binary names
- Supports standard (full) and minimal modes
- Preserves all customization comments

## Content Generation Rules

- **NEVER** add Claude Code attribution or co-authorship

### Phase 4: Auto-Detection & Substitution

**Detection Function Library** (bash functions to extract project values):

```bash
# Function 1: Extract project name from go.mod
detect_project_name() {
    local module_path=$(grep '^module ' go.mod 2>/dev/null | awk '{print $2}')
    [[ -z "$module_path" ]] && return 1
    # Strip version suffix and get basename
    basename "$module_path" | sed 's|/v[0-9]*$||'
}

# Function 2: Extract owner from git remote URL
detect_git_owner() {
    local remote_url=$(git remote get-url origin 2>/dev/null)
    [[ -z "$remote_url" ]] && return 1

    # SSH: git@github.com:sgaunet/repo.git → sgaunet
    if [[ "$remote_url" =~ git@[^:]+:([^/]+)/ ]]; then
        echo "${BASH_REMATCH[1]}"
        return 0
    fi

    # HTTPS: https://github.com/sgaunet/repo.git → sgaunet
    if [[ "$remote_url" =~ https?://[^/]+/([^/]+)/ ]]; then
        echo "${BASH_REMATCH[1]}"
        return 0
    fi

    return 1
}

# Function 3: Detect main package directory
detect_main_dir() {
    local main_files=$(find . -type f -name "main.go" 2>/dev/null | grep -v vendor | sort)
    [[ -z "$main_files" ]] && echo "." && return 0

    # Prefer cmd/* paths
    local cmd_main=$(echo "$main_files" | grep -E '^\./cmd/' | head -1)
    if [[ -n "$cmd_main" ]]; then
        dirname "$cmd_main" | sed 's|^\./||'
        return 0
    fi

    # Use first main.go found
    dirname "$(echo "$main_files" | head -1)" | sed 's|^\./||' | sed 's|^\.$|.|'
}

# Function 4: Detect registry from git platform
detect_registry() {
    local remote_url=$(git remote get-url origin 2>/dev/null)
    case "$remote_url" in
        *github.com*) echo "ghcr.io" ;;
        *gitlab.com*) echo "registry.gitlab.com" ;;
        *) echo "ghcr.io" ;;  # Default
    esac
}

# Function 5: Extract author info from git config
detect_author() {
    local name=$(git config user.name 2>/dev/null)
    local email=$(git config user.email 2>/dev/null)
    echo "${name}|${email}"
}

# Function 6: Substitute placeholders in content
substitute_placeholders() {
    local content="$1"
    local project_name="$2"
    local owner="$3"
    local registry="$4"
    local main_dir="$5"
    local author_name="$6"
    local author_email="$7"

    # Escape special characters for sed
    project_name=$(echo "$project_name" | sed 's/[&\/]/\\&/g')
    owner=$(echo "$owner" | sed 's/[&\/]/\\&/g')
    registry=$(echo "$registry" | sed 's/[&\/]/\\&/g')
    main_dir=$(echo "$main_dir" | sed 's/[&\/]/\\&/g')
    author_name=$(echo "$author_name" | sed 's/[&\/]/\\&/g')
    author_email=$(echo "$author_email" | sed 's/[&\/]/\\&/g')

    # Replace all placeholders (using | delimiter to avoid URL conflicts)
    content=$(echo "$content" | sed "s|\[PROJECT_NAME\]|${project_name}|g")
    content=$(echo "$content" | sed "s|\[PROJECT_BINARY\]|${project_name}|g")
    content=$(echo "$content" | sed "s|\[PROJECT_USER\]|${project_name}|g")
    content=$(echo "$content" | sed "s|\[OWNER\]|${owner}|g")
    content=$(echo "$content" | sed "s|\[GITHUB_USERNAME\]|${owner}|g")
    content=$(echo "$content" | sed "s|\[REGISTRY_URL\]|${registry}|g")
    content=$(echo "$content" | sed "s|\[MAIN_DIR\]|${main_dir}|g")
    content=$(echo "$content" | sed "s|\[AUTHOR_NAME\]|${author_name}|g")
    content=$(echo "$content" | sed "s|\[AUTHOR_EMAIL\]|${author_email}|g")

    echo "$content"
}
```

**Detection Workflow:**

1. **Detect all values:**
   ```bash
   PROJECT_NAME=$(detect_project_name)
   OWNER=$(detect_git_owner)
   REGISTRY=$(detect_registry)
   MAIN_DIR=$(detect_main_dir)
   AUTHOR_INFO=$(detect_author)
   AUTHOR_NAME=$(echo "$AUTHOR_INFO" | cut -d'|' -f1)
   AUTHOR_EMAIL=$(echo "$AUTHOR_INFO" | cut -d'|' -f2)
   ```

2. **Handle missing critical values** (use AskUserQuestion if empty):
   - If `PROJECT_NAME` is empty: Prompt user for project name or use "app"
   - If `OWNER` is empty: Prompt user for owner or preserve `[OWNER]` placeholder
   - Non-critical values (AUTHOR_NAME, AUTHOR_EMAIL) can be empty strings

3. **Substitute placeholders in all templates:**
   ```bash
   goreleaser_template=$(substitute_placeholders "$goreleaser_template" \
       "$PROJECT_NAME" "$OWNER" "$REGISTRY" "$MAIN_DIR" "$AUTHOR_NAME" "$AUTHOR_EMAIL")

   dockerfile_template=$(substitute_placeholders "$dockerfile_template" \
       "$PROJECT_NAME" "$OWNER" "$REGISTRY" "$MAIN_DIR" "$AUTHOR_NAME" "$AUTHOR_EMAIL")

   passwd_template=$(substitute_placeholders "$passwd_template" \
       "$PROJECT_NAME" "$OWNER" "$REGISTRY" "$MAIN_DIR" "$AUTHOR_NAME" "$AUTHOR_EMAIL")
   ```

**Fallback Strategy:**
- If detection fails for critical values: Use placeholders and warn in success report
- Graceful degradation: Preserve `[PLACEHOLDER]` if user skips prompt

## Mode Comparison

### Standard Mode (Default)

**Use when:**
- Publishing Docker images
- Need multi-architecture container support
- Want complete release automation
- Deploying to Kubernetes or container platforms

**Creates:**
- `.goreleaser.yml` - Full configuration
- `Dockerfile` - Multi-stage optimized build
- `resources/etc/passwd` - Non-root user config

**Benefits:**
- Complete Docker integration
- Security-hardened containers
- Minimal image size
- Multi-platform support

### Minimal Mode (`--minimal`)

**Use when:**
- Binary-only releases (no containers)
- Custom Docker setup exists
- Want minimal configuration
- Testing GoReleaser for first time

**Creates:**
- `.goreleaser.yml` only

**Benefits:**
- Simpler setup
- No Docker dependencies
- Easy customization
- Faster generation

## Best Practices

**Configuration maintenance:**
- Keep GoReleaser version updated (v2.0.0+)
- Test with `--snapshot` before tagging
- Review changelog generation settings
- Verify build matrix for target platforms

**Release workflow:**
- Use semantic versioning (v1.2.3)
- Write conventional commits for changelog
- Tag releases from stable branches
- Test multi-arch builds locally when possible

**Docker best practices:**
- Use specific base image versions
- Minimize layer count
- Run as non-root user
- Scan images for vulnerabilities

**CI/CD integration:**
- Store tokens in CI/CD secrets
- Run release only on tags
- Cache Docker layers
- Verify checksums in release artifacts

**Security considerations:**
- Sign releases with GPG/cosign
- Generate SBOMs for compliance
- Use attestation for supply chain security
- Keep base images updated

## Limitations & Caveats

**Project requirements:**
- Go modules required (go.mod must exist)
- Git repository required
- Semantic version tags expected (v1.2.3)

**Docker limitations:**
- Requires Docker with buildx for multi-arch
- Registry credentials needed for pushing
- Build times increase with multiple platforms
- ARM builds may need QEMU on x86 hosts

**Platform support:**
- CGO disabled by default (pure Go only)
- Windows ARM support limited
- Some platforms may need custom ldflags

**Configuration complexity:**
- Template has opinionated defaults
- May need customization for monorepos
- Homebrew tap requires separate repository

## Error Recovery

**If goreleaser check fails:**
1. Verify YAML syntax: `yamllint .goreleaser.yml`
2. Check for required fields (project_name, builds)
3. Validate file paths in configuration
4. Review error message for specific issues

**If builds fail:**
1. Check main package location: `go build <mainDir>`
2. Verify Go module is valid: `go mod tidy`
3. Test individual platforms: `goreleaser build --snapshot --single-target`
4. Review ldflags for syntax errors

**If Docker build fails:**
1. Test Dockerfile locally: `docker build -t test .`
2. Verify buildx is available: `docker buildx ls`
3. Check registry authentication
4. Try building for single platform first

**If release fails:**
1. Verify Git tag exists: `git tag -l`
2. Check token permissions (GitHub/GitLab)
3. Review network connectivity
4. Check for existing release with same tag
