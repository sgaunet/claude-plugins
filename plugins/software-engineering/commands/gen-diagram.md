# Generate d2 Diagram Command

Produce a `.d2` source file (and a rendered `.svg` when the `d2` binary is installed) that captures a software or infrastructure architecture. Default engine is **d2** (https://d2lang.com/), default icons come from **https://icons.terrastruct.com/**.

## Why This Command Exists

**Problem**: Architecture diagrams in design docs and READMEs are usually either missing, drawn in a closed-source tool that no one can edit, or pasted as PNG screenshots that drift from reality. Running through d2 syntax and the icon catalog by hand takes 10-20 minutes per diagram.

**Solution**: One command that:
- Synthesizes a `.d2` source file from a free-text description or repo analysis
- Embeds vendor icons from `icons.terrastruct.com` only when the URL is known to resolve
- Renders to SVG automatically if `d2` is installed
- Stores both files in `docs/diagrams/` for diff-able, PR-reviewable diagrams

## Content Generation Rules

**CRITICAL**: When generating any content (.d2 source, file headers, descriptions, or any other output):
- **NEVER** add Claude Code attribution or co-authorship
- **NEVER** include phrases like "Generated by Claude", "Created with Claude Code", or similar
- **NEVER** add "Co-Authored-By: Claude" in commit messages or documentation
- All generated content should appear as if written by a human developer
- Focus on technical accuracy and clear, readable d2 source

## Process (6 Phases)

### Phase 1: Prerequisites Validation

1. **Validate Git repository:**
   ```bash
   git rev-parse --git-dir
   ```
   - If fails: Exit with error "Not a git repository. Run 'git init' first."

2. **Detect d2 binary** (non-blocking):
   ```bash
   command -v d2
   ```
   - If found: Store `d2Installed = true`. Capture version: `d2 --version`.
   - If not found: Store `d2Installed = false` (will warn later, but proceed).

3. **Verify command assets:**
   - Use Read to confirm `${CLAUDE_PLUGIN_ROOT}/commands/assets/gen-diagram/d2-cheatsheet.md` exists.
   - If missing: Exit with error "Cheatsheet not found at ${CLAUDE_PLUGIN_ROOT}/commands/assets/gen-diagram/d2-cheatsheet.md. Plugin may be corrupted. Reinstall the software-engineering plugin."

4. **Resolve repository root** for output path defaults:
   ```bash
   git rev-parse --show-toplevel
   ```
   Store as `repoRoot`.

### Phase 2: Argument Parsing

Parse `$ARGUMENTS` (or `$argument`, depending on host syntax). Tokenize on spaces, then partition into flags and a free-text remainder.

**Supported flags:**
- `--type=<arch|sequence|er|deployment|c4|pipeline>` — diagram style. Default: inferred from description, falls back to `arch`.
- `--no-render` — skip the SVG render step even if d2 is installed.
- `--output=<path>` — explicit target path for the `.d2` file. Default: `docs/diagrams/<slug>.d2`.
- `--dry-run` — produce the preview only; do not write any files.

**Everything else** is joined back with spaces as `description`.

**Slugification:**
- If `description` is non-empty: lowercase, replace non-alphanumerics with `-`, collapse repeats, trim leading/trailing `-`, cap at 60 chars.
- If `description` is empty: slug = `architecture`.

**Type inference** (when `--type` is omitted):
- Description matches `/sequence|flow|step.*step|request.*response/i` → `sequence`
- Matches `/er|entity|schema|table/i` → `er`
- Matches `/deploy|infra|vpc|cluster|kubernetes|aws|gcp|azure/i` → `deployment`
- Matches `/c4|context|container|component/i` → `c4`
- Matches `/pipeline|ci\/cd|workflow|jobs/i` → `pipeline`
- Otherwise → `arch`

**Default output path:**
- If `--output` provided: use it verbatim (resolve relative to cwd).
- Else: `<repoRoot>/docs/diagrams/<slug>.d2`.

The accompanying SVG path is the same path with `.d2` replaced by `.svg`.

### Phase 3: Context Gathering

1. **Read the cheatsheet** to ground the LLM in real d2 syntax and verified icon URLs:
   ```
   Read: ${CLAUDE_PLUGIN_ROOT}/commands/assets/gen-diagram/d2-cheatsheet.md
   ```

2. **If `description` is empty**, launch one Explore subagent to summarize the repo's architecture:
   ```
   Task(
     subagent_type: "Explore",
     description: "Repo architecture summary",
     prompt: "
       Summarize this repository's architecture for a system diagram. Return:
       {
         entryPoints: [list of binaries / servers / endpoints discovered],
         services: [list of internal services or modules],
         datastores: [databases, caches, queues with vendor names],
         externalDeps: [third-party APIs, SaaS],
         deploymentTargets: [k8s, AWS service, etc., if obvious from IaC],
         language: dominant language,
         shape: 'monolith' | 'modular' | 'microservices' | 'library' | 'cli'
       }

       Detection strategy:
       - Languages: glob for *.go, *.py, *.js, *.ts, *.rs, *.java
       - Entry points: cmd/*/main.go, main.py, src/index.{ts,js}, bin/*
       - Datastores: search go.mod / package.json / requirements.txt for postgres, mysql, redis, kafka, mongo, sqlite, elasticsearch
       - Deployment: presence of Dockerfile, k8s/, helm/, *.tf, .github/workflows/

       Return concise JSON, no prose.
     "
   )
   ```
   Use the returned summary as the modeling input. Otherwise, skip this step.

3. **If `description` is non-empty**, treat it as the modeling input directly. Use the cheatsheet patterns to expand abbreviations (e.g. "auth flow with JWT and Redis" → sequence diagram with user, SPA, API, Redis cache).

### Phase 4: Confirmation Preview

Build the preview message:

```
📐 d2 Diagram Preview

Type:        <type>
Description: <description or "(auto-detected from repo)">
Output:      <relative path>.d2
Render:      <"yes (d2 → SVG)" if d2Installed && !--no-render else "no (source only)">

Planned content:
  • Layout: <direction> / <layout-engine>
  • Nodes (~N): <comma-separated short list>
  • Icons: <count of icons.terrastruct.com URLs that will be embedded>

<CONDITIONAL WARNINGS>
```

**Conditional warnings:**

If `d2Installed = false`:
```
ℹ️  d2 binary not found
    .d2 source will be created, but rendering is skipped.
    Install:
      • macOS:  brew install d2
      • Linux:  curl -fsSL https://d2lang.com/install.sh | sh -s --
      • Verify: d2 --version
```

If target `.d2` file already exists:
```
⚠️  WARNING: <path>.d2 already exists
    This will OVERWRITE the existing diagram source.
    Consider committing or backing up the current version first.
```

**Confirmation logic:**

If `--dry-run`:
```
[DRY RUN MODE - No files will be written]
```
Proceed straight to Phase 6 to print the preview and the would-be `.d2` content. Do **not** write or render.

If target `.d2` exists and not `--dry-run`:
```
Use AskUserQuestion:
  Question: "Overwrite existing diagram at <path>.d2?"
  Options:
    - "Yes, overwrite" (proceed to Phase 5)
    - "No, cancel" (exit with "Operation cancelled by user")
```

If target does not exist and not `--dry-run`:
```
Use AskUserQuestion:
  Question: "Create diagram at <path>.d2?"
  Options:
    - "Yes, create" (proceed to Phase 5)
    - "No, cancel" (exit with "Operation cancelled by user")
```

### Phase 5: Generation

1. **Synthesize the `.d2` source** following the cheatsheet:
   - Begin with a 1-3 line comment header: `# <Title>` and a sentence on scope/audience.
   - For `sequence`: start the body with `shape: sequence_diagram`.
   - For `er`: each table is a `shape: sql_table` block.
   - For `deployment`: nest containers by trust boundary (region → VPC → subnet).
   - For `c4`: use `shape: c4-person` for actors and `shape: package` for containers.
   - For `arch` / `pipeline`: pick `direction: right` and label every edge.
   - **Only include `icon:` URLs that appear in the cheatsheet's verified list.** If the design needs an icon not in the list, omit it (the user can add it manually) rather than guessing.
   - Apply layout engine `dagre` by default. Switch to `elk` if node count > 10.
   - Keep node count ≤ 12 per top-level view.

2. **Ensure parent directory exists:**
   ```bash
   mkdir -p "$(dirname <output>)"
   ```

3. **Write the file** with Write to the resolved output path.

### Phase 6: Render & Success Report

1. **Render to SVG** when `d2Installed && !--no-render && !--dry-run`:
   ```bash
   d2 <output>.d2 <output>.svg
   ```
   Capture exit code. If non-zero, report the d2 error to the user but treat as a soft failure (the `.d2` source was still produced).

2. **Verify written files:**
   ```bash
   test -f <output>.d2 && wc -l <output>.d2
   test -f <output>.svg  # only if render attempted
   ```

3. **Success report (normal execution):**
   ```
   ✅ Diagram created

   Files:
     ✓ <output>.d2 (<N> lines)
     ✓ <output>.svg              # only if render succeeded

   Type:        <type>
   Layout:      <direction> / <layout-engine>
   Icons used:  <count>

   Next steps:

     1. Preview the SVG:
        open <output>.svg                 # macOS
        xdg-open <output>.svg             # Linux

     2. Edit the source:
        ${EDITOR:-vi} <output>.d2

     3. Re-render after edits:
        d2 <output>.d2 <output>.svg

     4. Try a different layout:
        d2 --layout=elk <output>.d2 <output>.svg

     5. Embed in docs:
        ![Architecture](<relative-path>.svg)

     6. Browse the icon catalog:
        https://icons.terrastruct.com/

     7. d2 reference:
        https://d2lang.com/tour/intro
   ```

4. **Success report (dry-run):**
   ```
   ✅ [DRY RUN] Preview only — no files written

   Would create:
     • <output>.d2 (~<N> lines)
     • <output>.svg               # only if d2Installed && !--no-render

   --- Preview of <output>.d2 ---
   <full d2 source>
   --- end preview ---

   Run again without --dry-run to write.
   ```

5. **If `d2Installed = false`**, append:
   ```

   ⚠️  Install d2 to render automatically next time:
        brew install d2
        # or: curl -fsSL https://d2lang.com/install.sh | sh -s --
   ```

6. **If `d2` render failed**, append:
   ```

   ⚠️  d2 render failed (.d2 source was written successfully):
        <stderr from d2>
        Try manually: d2 <output>.d2 <output>.svg
   ```

## Integration with Other Commands

- **Before `/gen-diagram`**: none required.
- **After `/gen-diagram`**:
  - `/gen-claude` will pick up the new `docs/diagrams/` directory in subsequent runs.
  - Reference the SVG from generated docs or PR descriptions.
- **Companion agent**: invoking `/gen-diagram` proactively pulls in the `diagram-architect` agent for ongoing edits to `.d2` files in the repo.

## Examples

```bash
# Architecture from a description, render SVG if d2 is installed
/gen-diagram simple web app with frontend, API, postgres, redis

# Sequence diagram for an auth flow, custom output path
/gen-diagram --type=sequence --output=docs/diagrams/login.d2 login flow with JWT

# ER diagram from existing schema description
/gen-diagram --type=er users have many orders, orders have many line items

# Auto-discover from the repo (no description)
/gen-diagram

# Preview only, don't write
/gen-diagram --dry-run kafka-fronted ingestion service

# Skip rendering even when d2 is installed
/gen-diagram --no-render deployment topology on AWS
```

## Limitations & Caveats

- **d2 binary required for SVG**: only the source file is produced when `d2` is missing.
- **Icon catalog drift**: only icons listed in the cheatsheet are guaranteed to resolve. Add more by browsing https://icons.terrastruct.com/ and updating the cheatsheet.
- **Repo analysis is best-effort**: when `description` is empty, the Explore subagent summarizes what it can find — clean up the resulting diagram by hand for unusual layouts.
- **No PNG/PDF**: only SVG is rendered by default. Use `d2 --target=png` manually for raster output.
- **Node count cap**: diagrams with more than ~12 top-level nodes get crowded. Split into multiple files (`-context.d2`, `-containers.d2`) rather than fighting the layout.

## Error Recovery

**If `d2 --version` errors or `d2` rendering fails:**
1. Check version: `d2 --version` (require ≥ 0.6).
2. Try a different layout: `d2 --layout=elk <file>.d2`.
3. Validate syntax by reading the d2 stderr — it usually points at the offending line.

**If an icon URL 404s:**
1. Open https://icons.terrastruct.com/ and confirm the exact path (spaces become `%20`).
2. Either fix the URL in the `.d2` file or remove the `icon:` attribute and the surrounding `shape: image`.

**If the diagram is too dense:**
1. Re-run with `--type=c4` to layer it.
2. Split into separate files per concern (auth, billing, ingestion, …).
3. Increase grouping depth so siblings share a parent container.
